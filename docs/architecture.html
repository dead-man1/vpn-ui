Per-Client Traffic Tracking

1 Session Recording

When pppd authenticates a user via CHAP, the ip-up.d/l2tp-acct script runs:

# Lookup email from usermap USERNAME=$PEERNAME EMAIL=$(awk -v u="$USERNAME" \ '$1==u {print $2}' /etc/x-ui/l2tp-usermap) # Record session echo "$EMAIL $REMOTE_IP $IFACE" \ >> /etc/x-ui/l2tp-sessions # Add nft named counters + rules IP=$(echo $REMOTE_IP | tr '.' '_') nft add counter ip vpn l2tp_up_$IP nft add counter ip vpn l2tp_down_$IP nft add rule ip vpn l2tp_acct \ ip saddr $REMOTE_IP counter name l2tp_up_$IP nft add rule ip vpn l2tp_acct \ ip daddr $REMOTE_IP counter name l2tp_down_$IP

2 Counter Collection

Every 10 seconds, XrayTrafficJob calls NftService.CollectAndResetTraffic():

# Atomic read+reset all counters (JSON) nft -j reset counters table ip vpn => l2tp_up_10_0_2_10: 1140 bytes => l2tp_down_10_0_2_10: 1222 bytes # Read session files for IP->email map l2tp: readSessionFile("/etc/x-ui/l2tp-sessions") pptp: readSessionFile("/etc/x-ui/pptp-sessions") # Map counter name -> IP -> email return l2tpTraffics, pptpTraffics

3 Database Update

L2TP traffic merges into the existing Xray traffic pipeline:

# In XrayTrafficJob.Run(): xrayTraffics = GetXrayTraffic() l2tp, pptp = nft.CollectAndResetTraffic() # Merge into single pipeline clientTraffics = append( xrayTraffics, l2tp..., pptp...) # Update DB (accumulative) AddTraffic(traffics, clientTraffics) => client_traffics.up += delta.up => client_traffics.down += delta.down => client_traffics.all_time += total => client_traffics.last_online = now
